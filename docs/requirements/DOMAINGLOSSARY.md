# Domain Glossary

Use these terms consistently across code, documentation, API contracts, and UI labels.

| Term | Definition |
|---|---|
| Actor | A person associated with a movie cast entry. |
| Admin | A user with `ROLE_ADMIN`, allowed to manage movie metadata, upload videos, inspect encoding status, and retry failed encoding jobs. |
| Backend | The Spring Boot core API runtime. It owns auth, domain APIs, validation, persistence, upload handling, and queue publishing. |
| Continue Watching | A personalized list of movies the user started but has not finished, ordered by recent progress. |
| Director | A person associated with directing a movie. |
| Encoding Job | A RabbitMQ message instructing the worker to process a video asset into HLS output. |
| Encoding Status | The lifecycle state of a video asset: `UPLOADED`, `QUEUED`, `PROCESSING`, `READY`, or `FAILED`. |
| FFmpeg | Command-line tool used by the worker to transcode video into HLS. |
| FFprobe | Command-line tool used by the worker to inspect media metadata such as duration, codec, bitrate, and resolution. |
| Genre | A category assigned to movies for browsing, filtering, and search. |
| HLS | HTTP Live Streaming, the MVP playback format based on `.m3u8` playlists and segment files. |
| HLS Artifact | A generated playlist or media segment produced by FFmpeg and stored in MinIO. |
| Master Playlist | The main `.m3u8` playlist returned for playback. For MVP it may point to one quality level. |
| MinIO | S3-compatible object storage used for raw video files, HLS artifacts, thumbnails, and optional subtitles. |
| Movie | The catalog entity users browse, search, watch, rate, and add to a watchlist. |
| Movie Metadata | Descriptive movie data such as title, description, year, genre, actor, director, rating summary, and video availability. |
| Nginx | Reverse proxy and static delivery layer used for frontend, API proxying, and HLS delivery headers where applicable. |
| Playback Progress | The user's saved current playback position for a movie. |
| PostgreSQL Full-Text Search | MVP search mechanism for matching title, description, genre, actor, and director metadata. |
| RabbitMQ | Message broker used for encoding job handoff from backend to worker. |
| Rating | A user-submitted score for a movie. Each user can create or update their own rating. |
| Raw Video | The originally uploaded video file before transcoding. |
| Redis | Cache and temporary buffer for homepage/search data and playback progress before PostgreSQL sync. |
| Resume Playback | Starting playback near the user's last saved playback position. |
| Role | Authorization label assigned to a user, initially `ROLE_USER` or `ROLE_ADMIN`. |
| Segment | A media chunk generated by FFmpeg for HLS playback, usually a `.ts` file for MVP. |
| Subtitle | Optional timed text track, expected as WebVTT if introduced after the base MVP. |
| Thumbnail | A still image extracted from a video and shown in catalog UI; recommended after base MVP works. |
| User | A registered account with `ROLE_USER`, allowed to browse, watch, resume, manage watchlist/history, and rate movies. |
| User Profile | The current user's account-facing profile data, including display name and read-only email. |
| Video Asset | The technical media record linked to a movie, including raw object location, HLS output location, metadata, and encoding status. |
| Watch History | A user's chronological record of watched or started movies. |
| Watchlist | A user's saved list of movies to watch later. |
| Worker | The Spring Boot video-processing runtime responsible for consuming jobs, running FFprobe/FFmpeg, uploading HLS output, and updating status. |

## Reserved Status Values

| Status | Meaning |
|---|---|
| `UPLOADED` | Raw file exists and a video asset record has been created. |
| `QUEUED` | An encoding job has been published and is waiting for worker processing. |
| `PROCESSING` | Worker is actively probing or transcoding the asset. |
| `READY` | HLS output is available for playback. |
| `FAILED` | Processing failed and requires admin inspection or retry. |

## Naming Rules

- Use `movie` for catalog content shown to users.
- Use `video asset` for technical media records and processing state.
- Use `raw video` for the original upload.
- Use `HLS artifact` for generated streaming files.
- Use `worker` for the processing runtime; do not call it a microservice.
