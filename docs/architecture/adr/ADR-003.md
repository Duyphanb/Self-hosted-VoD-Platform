# ADR-003: PostgreSQL With Flyway As Schema Authority

Status: Frozen for MVP implementation

## Status

Accepted

## Context

The MVP needs durable relational data for users, roles, movies, playback progress, watchlist, history, ratings, video assets, and encoding job status. Search must use real metadata.

## Decision

Use PostgreSQL as the source-of-truth database and Flyway for schema migrations.

MVP search uses PostgreSQL full-text search with a GIN index.

## Options Considered

| Option | Pros | Cons |
|---|---|---|
| PostgreSQL + Flyway | Strong relational model, FTS support, migration discipline | Requires schema planning |
| MySQL | Familiar relational option | Weaker built-in FTS fit for this project |
| Elasticsearch | Powerful search | Out of scope and too heavy for MVP |
| NoSQL primary store | Flexible documents | Poor fit for relational ownership and RBAC |

## Consequences

- ERD must be reviewed before implementation.
- Existing committed migrations must not be edited.
- Query code must use parameterized ORM/query mechanisms.
- Search can start simple and remain explainable.
