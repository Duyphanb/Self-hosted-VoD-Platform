# ADR-002: Java 21, Spring Boot 3.x, Maven Multi-Module

Status: Frozen for MVP implementation

## Status

Accepted

## Context

The backend and worker need a maintainable Java foundation that supports REST APIs, validation, security, persistence, testing, and operational health checks.

## Decision

Use Java 21, Spring Boot 3.x, and Maven multi-module structure for backend and worker development.

## Options Considered

| Option | Pros | Cons |
|---|---|---|
| Java 21 + Spring Boot 3.x | Modern LTS Java, strong ecosystem, good portfolio signal | Requires disciplined module boundaries |
| Node.js backend | Same language as frontend | Weaker alignment with Java backend portfolio goal |
| Separate repositories | Cleaner isolation | More overhead for a solo portfolio project |

## Consequences

- Backend and worker can share Java conventions and test tooling.
- Spring Security, Spring Data JPA, Validation, Actuator, and Springdoc fit the MVP.
- Maven modules must stay simple and not become a premature enterprise framework.
